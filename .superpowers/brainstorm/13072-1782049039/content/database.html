📋 核心表说明

1. users — 用户表

字段 类型 说明
email, phone, password_hashVARCHAR邮箱/手机号登录 + 密码哈希
wechat_unionid, qq_openidVARCHAR微信/QQ 登录预留，可空
roleENUMUSER（普通用户）| ADMIN（管理员）

2. posts — 分类信息表（核心）

字段 类型 说明
typeENUMJOB / SEEK / SHOP_TRANSFER / SECONDHAND / TRAINING / MODEL / BOOKING / SERVICE
category_idFK关联分类，可扩展到子分类
city_idFK所属城市
price_min, price_maxINT薪资范围/转让价格
is_pinned, pin_type, pin_expires_atBOOLEAN/ENUM/DATE付费置顶：LARGE_TOP / MEDIUM_TOP / SMALL_TOP + 过期时间
statusENUMPENDING / PUBLISHED / REJECTED / EXPIRED
expires_atDATE信息有效期，过期自动下架

3. businesses / artists — 商家和纹身师黄页

商家（纹身店）和纹身师是独立的表，businesses 存储店铺信息（名称/Logo/电话/地址），artists 存储纹身师信息（可关联到店）。

4. articles — 新闻资讯

文章支持分类（公告/知识/趣闻/大师），通过 category_id 关联。使用 Markdown 编辑器（如 TipTap）。

5. qa_questions / qa_answers — 纹身问答

简化的问答系统：用户提问 → 回答 → 标记最佳答案。类似 Stack Overflow 但更简单。

6. payments — 支付订单（预留）

字段 说明
order_no, amount, pay_method, status订单号/金额/支付方式/状态
post_id, user_id, pin_type, pin_days关联的信息/用户/置顶类型/天数

📊 数据量预估

参考原站数据量：45页信息列表 × 每页约40条 ≈ 1800+条活跃信息，100+ 纹身店，100+ 篇文章

PostgreSQL 单表百万级以下完全无压力。初期无需分库分表，合理加索引即可。

🔑 关键索引策略

-- 高频查询索引
posts(city_id, type, status, created_at DESC)    -- 城市+类型筛选
posts(category_id, status)                        -- 分类筛选
posts(user_id, status)                            -- 用户信息管理
posts(is_pinned, pin_expires_at)                  -- 置顶排序
businesses(city_id, is_verified)                  -- 城市商家查询
articles(category_id, is_published, created_at)   -- 文章列表
qa_questions(is_resolved, created_at DESC)        -- 问答列表
