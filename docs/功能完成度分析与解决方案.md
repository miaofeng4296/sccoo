### 4. 🟡 找回密码邮件发送（中优先级）

**当前状态：**
- `/api/forgot-password` — 生成 reset token，但只 `console.log` 到终端
  ```typescript
  console.log(`[PASSWORD RESET] User: ${email}, Token: ${token}`);
  console.log(`[PASSWORD RESET] Reset URL: http://localhost:3000/reset-password?token=...`);
  ```
- `/api/user/reset-password` — **不验证 token**，有邮箱就直接改密码（安全漏洞）
- Token 未存储到数据库，无过期机制
- 前端 `/reset-password` 页面让用户手动输入邮箱

**缺失：**
- 邮件发送服务（SMTP/SendGrid/阿里云邮件推送）
- Token 存储与验证机制
- Token 过期处理
- 重置密码页面应通过 URL 参数传递 token

**解决方案：**
```
1. DB 添加 PasswordResetToken 表（或给 User 加 resetToken/resetTokenExpires 字段）
2. 接入邮件服务：
   - 开发测试：Ethereal（免费 fake SMTP）
   - 生产：阿里云邮件推送 / SendGrid / Resend
3. 安全修复：
   - /api/forgot-password POST → 生成 token 存 DB + 发送邮件
   - /api/user/reset-password POST → 验证 token + 邮箱 + 修改密码 + 删除 token
   - Token 有效期 1 小时
4. 前端 /reset-password → 从 URL 读取 token 和 email
```

**涉及文件：**
- `apps/web/src/app/api/forgot-password/route.ts`
- `apps/web/src/app/api/user/reset-password/route.ts`
- `apps/web/src/app/forgot-password/page.tsx`
- `apps/web/src/app/reset-password/page.tsx`
- `packages/db/prisma/schema.prisma`

## 四、技术债务备注

1. **Prisma 连接** — 每个 API 路由都直接 `import { prisma }` 而不是通过单例，但 Prisma 官方推荐的做法在 Next.js 中需要注意 hot reload 问题（目前靠 `@sccoo/db` 包的 module resolution 可能有问题）
2. **Next.js 16 兼容性** — 项目使用了 Next.js 16 Beta，API 和约定可能和文档有差异，升级生产版时需注意
3. **TypeScript 类型安全** — auth.ts 中大量使用 `(token as any)` 类型断言，应使用 `auth-types.ts` 进行模块扩展
4. **payment 价格计算逻辑有 bug** — `payments/route.ts` 中 `amount = (prices[pinType] || 200) * (pinDays / 30)`，对于不同套餐（365天/180天/30天）应按总价计算而非按天
5. **小程序 API base URL** — 硬编码 `localhost:3000`，需改为环境变量
