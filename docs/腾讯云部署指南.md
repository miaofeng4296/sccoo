买完后在控制台 → 防火墙 → 添加规则：

## 三、一键部署

### 方式 1：用部署脚本（推荐）

在本地项目目录执行：

```bash
# Windows 用 Git Bash，Mac/Linux 用终端
bash deploy.sh 你的服务器IP sccoo.cn
```

脚本会自动完成：
- 上传代码到服务器
- 安装 Docker + Nginx
- 构建并启动应用
- 配置 Nginx 反向代理
- 申请 SSL 证书

### 方式 2：手动逐步（如果脚本报错）

#### 3.1 安装 Docker

```bash
# SSH 到服务器后执行
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# 安装 Docker Compose
apt-get update
apt-get install -y docker-compose-plugin
```

#### 3.2 上传项目

在**本地**执行：

```bash
# 打包项目（排除不需要的文件）
cd D:/DingDan/sccoo
tar --exclude='node_modules' \
    --exclude='.next' \
    --exclude='.turbo' \
    --exclude='.git' \
    --exclude='*.db' \
    -czf sccoo.tar.gz .

# 上传到服务器
scp sccoo.tar.gz root@你的IP:/opt/
```

然后**在服务器上**解压：

```bash
cd /opt
tar -xzf sccoo.tar.gz -C /opt/sccoo/
rm sccoo.tar.gz
```

#### 3.3 创建环境变量

```bash
cd /opt/sccoo

# 生成随机密钥
DB_PASSWORD=$(openssl rand -base64 16)
NEXTAUTH_SECRET=$(openssl rand -base64 32)

# 写入 .env 文件
cat > .env << EOF
DB_PASSWORD=${DB_PASSWORD}
NEXTAUTH_SECRET=${NEXTAUTH_SECRET}
NEXTAUTH_URL=https://你的域名或IP
EOF
```

> **重要：** 如果有域名填域名；如果只用 IP 访问，填 `http://你的服务器IP`

#### 3.4 构建并启动

```bash
docker compose --env-file .env up -d --build
```

第一次构建需要 **5-10 分钟**（下载镜像 + 安装依赖 + Next.js 编译）。后续更新只需 1-2 分钟。

#### 3.5 检查状态

```bash
docker compose ps
# 应该看到两个服务都是 Up 状态

docker compose logs app --tail 20
# 看 Next.js 是否启动成功

curl http://localhost:3000
# 能返回 HTML 就说明成功了
```

#### 3.6 配置 Nginx

```bash
apt-get install -y nginx

# 编辑配置
nano /etc/nginx/sites-available/sccoo
```

填入以下内容（把 `your-domain.com` 替换成你的域名或 IP）：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 10M;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
ln -sf /etc/nginx/sites-available/sccoo /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t && systemctl reload nginx
```

#### 3.7 配置 SSL（如果有域名）

```bash
apt-get install -y certbot python3-certbot-nginx
certbot --nginx -d your-domain.com
```

证书会自动续期（certbot 有 systemd timer）。

## 四、部署后的必要操作

### 4.1 初始化数据库 + 种子数据

```bash
# 进入 app 容器
docker compose exec app sh

# 在容器内执行：
cd packages/db
npx prisma db push          # 创建表结构
npx tsx prisma/seed.ts      # 导入种子数据（分类、城市等）

exit
```

### 4.2 创建管理员账号

方式一：Web 注册后用命令行改角色

```bash
docker compose exec postgres psql -U sccoo -d sccoo -c "UPDATE users SET role='ADMIN' WHERE email='你的邮箱';"
```

方式二：在 Prisma seed 中加一个默认 admin 账号：

```typescript
// packages/db/prisma/seed.ts
import { prisma } from '../src';
import { hash } from 'bcryptjs';

async function main() {
  // ... existing seed code ...

  // 创建默认 admin
  const adminPassword = await hash('admin123456', 10);
  await prisma.user.upsert({
    where: { email: 'admin@sccoo.cn' },
    update: {},
    create: {
      email: 'admin@sccoo.cn',
      passwordHash: adminPassword,
      name: '管理员',
      role: 'ADMIN',
    },
  });
}
```

### 4.3 配置 NEXTAUTH_URL

部署后 `NEXTAUTH_URL` 必须是实际访问地址，否则登录会失败：

```bash
# 如果是域名
NEXTAUTH_URL=https://sccoo.cn

# 如果只用 IP
NEXTAUTH_URL=http://43.156.12.34
```

修改后重启：

```bash
docker compose up -d --force-recreate
```

## 五、日常运维命令

```bash
cd /opt/sccoo

# 查看日志
docker compose logs -f app

# 查看数据库日志
docker compose logs -f postgres

# 重启应用
docker compose restart app

# 重启所有服务
docker compose restart

# 停止全部
docker compose down

# 开始全部
docker compose up -d

# 进入应用容器
docker compose exec app sh

# 进入数据库
docker compose exec postgres psql -U sccoo -d sccoo

# 备份数据库
docker compose exec postgres pg_dump -U sccoo sccoo > backups/backup_$(date +%Y%m%d_%H%M%S).sql

# 更新代码后重新部署
git pull
docker compose up -d --build
```

## 八、常见问题

### Q: 香港服务器国内访问慢吗？
A: 延迟 30-60ms，完全够用。比备案省 1-2 周时间。

### Q: 如果不想用 Docker？
A: 可以在服务器上直接装 Node.js 20.18 + PostgreSQL + pnpm，但 Docker 更省心（一条命令启动所有服务）。

### Q: 内存不够怎么办？
A: Next.js build 时会设置 `NODE_OPTIONS="--max-old-space-size=512"`。如果实在不够，可以在服务器上加 swap：
```bash
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
```

### Q: 图片上传存哪儿？
A: 目前的 Docker 部署是存本地。图片会自动放在 `apps/web/public/uploads/`。但容器重启会丢 — 建议尽快接入 Cloudflare R2（免费 10GB，下一期文档写）。

### Q: 怎么让服务开机自启？
A: Docker Compose 的 `restart: always` 已配置好。服务器重启后 Docker 会自动拉起容器。
